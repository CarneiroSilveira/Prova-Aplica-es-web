O que é um domínio e um DNS.

O Sistema de Nomes de Domínio (DNS) funciona quase como uma agenda de contatos no seu celular. Isso porque ele reduz a lacuna de comunicação entre computadores e humanos, ligando os nomes de domínio com os seus respectivos endereços de Protocolo de Internet (IP). E no caso o domínio seria o nome da genda de contatos. Ao digitar o nome do contato desejado ex: google.com na verdade o que o DNS por debaixo dos panos está fazendo é convertendo o nome google.com para o ip real do servidor que no caso seria 8.8.8.8

Explique o que é um servidor compartilhado e um servidor dedicado, evidenciando suas vantagens e desvantagens.

Um servidor dedicado é um servidor físico exclusivo para um único cliente ou organização. Vantagens: Possuem maior desenpenho, maior segurnaça. Pois no caso de um outro cliente instalar um software malisioso não só ele mais vc que não tem nada haver com ele também sera prejudicado, e maior controle sobre a maquina já que vc terá acesso ao root. Desvantagens: Custo geralmente esté tipo de serviço é mais caro e mais adequado para sites com alto tráfego ou aplicações críticas. Gerenciamento requer mais conhecimento técnico para manutenção e configuração. Com grandes poderes vem grandes responsabilidades

O que é um servidor de banco de dados, explique como fazer o backup e resstaurar esse backup. Utilize Imagesn para Ilustrar. para facilitar utilize o arquivo em anexo

Instalação e Configuração do MariaDB

Instala o servidor do banco de dados MariaDB

        sudo apt update
        sudo apt install mariadb-server
        sudo mysql_secure_installation

O mysql_secure_installation é rodado apenas na primeira vez durante a instalação.

Instala os conectores do banco no PHP

        sudo apt install php-mysql php-pdo

Verifica o status do servidor

        sudo service mariadb status

Inicia o servidor

        sudo service mariadb start

Para o servidor

        sudo service mariadb stop

Acessar o servidor

        sudo mysql -uroot -p

Criando um novo usuário

        CREATE USER 'senac'@'localhost' IDENTIFIED BY 'senac123';
        GRANT ALL PRIVILEGES ON * . * TO 'senac'@'localhost';
        FLUSH PRIVILEGES;
        exit

Acessa o servidor do MariaDB com o usuário criado

        mariadb -usenac -p

Cria um banco de dados

        create database webservices;

Marca um banco para ser utilizado

        use webservices;

Cria uma tabela de usuários

        create table usuarios (
            id int not null primary key auto_increment,
            login varchar(100) not null,
            nome varchar(100) not null,
            senha varchar(255) not null
        );

Exibe as tabelas criadas

        show tables;

Sair do servidor do banco

        exit

Inserir dados na tabela de usuários

        INSERT INTO usuarios (nome, login, senha) VALUES
        ('joao', 'João da Silva', '123456'),
        ('maria', 'Maria Antonieta', '456789');

Exporta um banco

        mysqldump -usenac -p webservices > backup.sql

Importa um banco

        mysql -usenac -p webservices_homologacao < backup.sql

Cria uma tabela post no banco webservice

        create table post (
            id int not null primary key auto_increment,
            titulo varchar(255) not null,
            descricao text not null
        );

Faz um novo backup do banco e importa na homologação

        mysqldump -usenac -p webservices > backup.sql
        mysql -usenac -p webservices_homologacao < backup.sql
